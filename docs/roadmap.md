# Roadmap

## 时间范围

当前规划覆盖 **2026-03-09** 到 **2026-06-08**。其中前半段用于方向收敛与基线部署，后半段再进入 C++ 自研系统的设计和实现阶段。

## Milestones

### M0: 初步调研

- 时间: `2026-03-09` - `2026-03-16`
- 内容:
  - 调研 eBPF、协程调度器、KV Cache 等方向
  - 比较系统性、可落地性和展示空间

### M1: 方向收敛

- 时间: `2026-03-16` - `2026-03-23`
- 内容:
  - 确定以 KV Cache 为主线
  - 继续调研 vLLM、llama.cpp 与 serving 系统

### M2: 主题细化

- 时间: `2026-03-23` - `2026-03-30`
- 内容:
  - 明确以 KV Cache 调度与生命周期管理为主要问题
  - 初步确定长期目标为可移植、可独立演进的 scheduler / runtime

### M3: 环境准备与早期验证

- 时间: `2026-03-30` - `2026-04-13`
- 内容:
  - 完成分工、资料整理和环境预备
  - 为 vLLM 本地 bring-up 做前置准备

### M4: vLLM 本地部署与 smoke test

- 时间: `2026-04-14` - `2026-04-27`
- 内容:
  - 在本地 Linux / WSL2 环境中部署官方 vLLM
  - 跑通 offline inference 与 online serving
  - 记录环境与最小可用验证结果
  - 在仓库内沉淀可复用的 `vllm_baseline/` 工作区

### M5: vLLM 代码路径梳理与基线测试

- 时间: `2026-04-28` - `2026-05-11`
- 内容:
  - 阅读 scheduler、prefix cache、paged attention、hybrid cache manager 等关键路径
  - 跑第一轮基线测试
  - 形成未来自研系统要继承和要突破的能力列表

### M6: C++ 架构冻结

- 时间: `2026-05-12` - `2026-05-18`
- 内容:
  - 明确未来 C++ runtime 的模块边界
  - 冻结 metadata、scheduler、backend abstraction 的第一版设计
  - 整理接口与文档

### M7: 自研实现阶段

- 时间: `2026-05-19` - `2026-05-25`
- 内容:
  - 进入主体代码实现
  - 争取在五月中下旬前完成核心模块的第一版

### M8: Benchmark、报告与展示

- 时间: `2026-05-26` - `2026-06-08`
- 内容:
  - 跑对比测试
  - 整理报告、图表和展示材料
  - 完成仓库清理与阶段总结
