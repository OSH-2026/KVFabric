# Quality Task Catalog

当前任务集文件位于：

`configs/quality_tasks_general_v1.json`

## 任务分类

### 1. Long Context QA

目标：

- 看长上下文里埋入的关键信息能否被稳定取出
- 看 candidate 方案是否更容易遗漏稀疏但关键的事实

当前条目：

- `long_qa_001`
- `long_qa_002`

### 2. Multi-turn Dialog

目标：

- 看多轮历史中是否还能保留用户偏好、时间、重点文件等信息
- 这类任务对“历史记忆被压缩后是否失真”比较敏感

当前条目：

- `dialog_001`
- `dialog_002`

### 3. Code Generation

目标：

- 看候选方案是否会让代码输出更短缺、更漏关键逻辑
- 通过必要关键字来做低成本自动检查

当前条目：

- `code_001`
- `code_002`

### 4. Reasoning

目标：

- 看结构化推理和最终答案是否更容易出错
- 当前采用明确数值题，便于规则打分

当前条目：

- `reason_001`
- `reason_002`

## 扩展建议

后续如果要增强这套任务集，建议优先按下面顺序扩展：

1. 每个类别先加样本数
2. 再加更长上下文版本
3. 再加更难的代码和推理任务
4. 最后再引入人工评审或更复杂自动评测
